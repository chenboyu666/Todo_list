# Windows Floating TODO 设计规格

日期：2026-05-12

## 1. 目标

在当前项目中实现一个 Windows 桌面浮动 TODO 程序。程序需要实时显示时间，始终浮于桌面之上，可拖动定位，可创建多个任务，并按优先级、工作量、截止时间、倒计时和手动进度展示。应用支持系统托盘后台运行、开机自启、Windows 通知，并可通过 PyInstaller 打包成一个可直接启动的文件夹。

第一版重点是做出稳定可用的本地桌面工具，而不是复杂协作平台或云同步应用。

## 2. 已确认决策

- 技术栈：Python + PySide6。
- 数据存储：第一版使用本地 JSON 文件。
- 发布方式：PyInstaller `onedir`，输出一个包含 exe、依赖动态库、资源和默认配置的文件夹。
- 主界面：任务面板型浮窗，而不是单任务计时器或横向停靠条。
- UI 风格：冷静科技任务控制台。深色、数据密集、状态色清晰、科技感克制。
- 倒计时规则：同时支持预计工作量和截止时间。
- 进度规则：进度由用户手动设置，不根据工作量自动计算。
- 后台能力：托盘后台、开机自启、临近截止通知、到期通知。

## 3. 非目标

第一版不做账号系统、云同步、多设备同步、团队协作、复杂报表、日历双向同步、插件系统或完整番茄钟。工作量字段用于展示和排序，不在第一版中驱动自动进度。

## 4. 架构

应用拆分为以下模块：

- `MainWindow`：桌面浮窗。负责实时显示时间、任务摘要、聚焦任务、任务队列、倒计时、手动进度、窗口拖动、锁定位置和置顶。
- `SettingsWindow`：设置界面。负责主题、透明度、窗口行为、开机自启、通知提前时间、低干扰模式和数据路径提示。
- `TaskStore`：本地 JSON 数据读写。负责加载、保存、校验任务和设置。
- `TimerService`：每秒刷新当前时间、任务截止倒计时、临近截止状态和到期状态。
- `NotificationService`：负责 Windows Toast 通知，处理临近截止和到期提醒。
- `TrayService`：系统托盘图标和菜单。支持显示/隐藏主窗、快速新增任务、打开设置和退出程序。
- `Packaging`：PyInstaller 打包配置与 `scripts/build.ps1` 构建脚本。

模块之间保持简单依赖：窗口层读取应用状态并发出用户操作，服务层更新状态，`TaskStore` 持久化数据。

## 5. UI 与交互

主浮窗采用“冷静科技任务控制台”布局：

- 顶部显示实时数字时钟、日期、置顶状态、锁定位置按钮和设置按钮。
- 摘要区显示今日完成率、进行中任务数量、临近截止数量。
- 聚焦任务区显示当前最高优先级或最紧急任务，包含优先级、任务名、预计工作量、截止时间、截止倒计时和手动进度。
- 任务队列按优先级、截止紧急度、工作量排序。每个任务显示 P1/P2/P3、任务名、工作量、截止倒计时和进度条。
- 新增/编辑任务界面支持任务名、优先级、预计工作量、截止时间、手动进度和备注。
- 设置窗口管理置顶、锁定位置、透明度、开机自启、关闭到托盘、通知提前时间和低干扰模式。

交互规则：

- 浮窗可拖动，锁定位置后不能拖动。
- 点击关闭按钮时默认隐藏到托盘，不退出进程。
- 托盘菜单包含显示/隐藏、快速新增任务、打开设置和退出。
- 图标按钮必须有 tooltip。
- 到期或临近截止时发送系统通知，并在主窗中对对应任务做轻微高亮。
- 动效只用于状态变化，例如进度变化、新增任务淡入、提醒状态。避免持续闪烁和装饰性循环动画。

## 6. UI 设计系统

根据 `ui-ux-pro-max` 检索结果，本项目使用实时运维/数据密集控制台风格，而不是高噪声赛博朋克风格。

颜色令牌：

```text
background:  #020617
panel:       #0E1223
surface:     #111827
surface-2:   #172033
border:      #334155
text:        #F8FAFC
muted-text:  #94A3B8
accent:      #22C55E
info:        #22D3EE
warning:     #F59E0B
danger:      #EF4444
```

视觉规则：

- 深色背景上保证正文对比度充足。
- 任务优先级不能只靠颜色表达，必须同时显示 `P1`、`P2`、`P3` 文本标签。
- 倒计时、时间和百分比使用等宽数字，避免每秒刷新造成视觉抖动。
- 科技感来自细边框、状态色、克制高光和整齐信息层级，不依赖大面积霓虹、故障动画或扫描线。
- 圆角控制在 8px 左右，面板紧凑但不拥挤。
- 交互反馈使用 150-250ms 的轻微过渡。
- 低干扰模式减少发光、脉冲和高亮，仅保留必要状态提示。

## 7. 数据模型

任务字段：

```json
{
  "id": "uuid",
  "title": "完成 Windows 浮动 TODO 的设计规格",
  "priority": "P1",
  "effort_minutes": 90,
  "deadline": "2026-05-12T23:00:00+08:00",
  "progress": 68,
  "status": "active",
  "created_at": "2026-05-12T00:00:00+08:00",
  "updated_at": "2026-05-12T00:00:00+08:00",
  "completed_at": null,
  "notes": "",
  "notification_state": {
    "deadline_warning_sent": false,
    "deadline_due_sent": false
  }
}
```

设置字段：

```json
{
  "always_on_top": true,
  "lock_position": false,
  "close_to_tray": true,
  "launch_on_startup": false,
  "opacity": 0.96,
  "low_distraction_mode": false,
  "notification_lead_minutes": 15,
  "window_geometry": {
    "x": 1200,
    "y": 120,
    "width": 410,
    "height": 620
  },
  "theme": "calm-tech-dark"
}
```

本地文件：

- `data/tasks.json`：任务列表。
- `data/settings.json`：设置。

写入时使用临时文件替换目标文件，降低意外关闭导致 JSON 损坏的风险。读取失败时保留损坏文件副本，并使用默认空数据启动。

## 8. 排序、聚焦与倒计时

排序规则：

1. `priority`：P1 高于 P2，高于 P3。
2. `deadline`：有截止时间且更临近的任务排前。
3. `effort_minutes`：工作量更大的任务排前。
4. `created_at`：更早创建的任务排前。

聚焦任务规则：

- 从 `status = active` 的任务中选择。
- 优先级最高、截止时间最临近、工作量更大的任务成为聚焦任务。
- 没有任务时显示空状态和新增任务入口。

倒计时规则：

- 有 `deadline` 的任务显示距离截止时间剩余多久。
- 截止时间已过时显示超时多久，并使用危险状态。
- `effort_minutes` 只展示预计工作量，不自动计算进度。
- `progress` 由用户手动设置，范围为 0-100。
- 进度到 100% 时提示用户是否标记完成，也允许用户手动完成任务。

## 9. 后台运行与通知

后台运行：

- 程序启动后显示主浮窗，并创建系统托盘图标。
- 关闭主窗时默认隐藏到托盘。
- 托盘菜单提供显示/隐藏主窗口、快速新增任务、打开设置、退出。
- 只有点击托盘菜单中的退出才结束进程。

开机自启：

- 设置中可开启或关闭开机自启。
- Windows 实现使用当前用户注册表路径：`HKCU\Software\Microsoft\Windows\CurrentVersion\Run`。
- 写入值为当前打包 exe 的完整路径。

通知：

- 使用 Windows Toast 通知。
- 默认提前 15 分钟发送临近截止提醒。
- 截止时间到达时发送到期提醒。
- 每个任务记录已提醒状态，避免重复提醒。
- 低干扰模式下不做主窗闪烁或脉冲，只保留系统通知和静态状态变化。

## 10. 打包发布

发布方式：

- 使用 PyInstaller `onedir` 模式。
- 输出目录为 `dist/FloatingTODO/`。
- 用户通过双击 `FloatingTODO.exe` 启动。
- 依赖动态库、资源、默认配置和数据目录放在同一发布文件夹中。

项目脚本：

- `scripts/build.ps1`：创建虚拟环境、安装依赖、运行 PyInstaller、生成发布文件夹。
- `requirements.txt`：记录 PySide6、通知相关依赖和打包依赖。
- `README.md`：说明运行方式、打包方式和数据位置。

数据默认放在发布目录下的 `data/`，便于整个文件夹复制和携带。后续如果需要多用户隔离或更符合 Windows 规范，可以增加选项改用 `%APPDATA%/FloatingTODO/`。

## 11. 错误处理

- JSON 读取失败：复制损坏文件为 `.broken-时间戳`，使用默认数据启动，并在设置界面显示提示。
- JSON 写入失败：保留内存状态，显示错误提示，下一次保存继续尝试。
- 通知发送失败：记录日志，不阻断主程序。
- 开机自启写入失败：在设置界面提示权限或路径问题。
- 托盘不可用：主窗口仍可正常运行，并在设置中显示托盘不可用提示。

## 12. 测试与验收

功能验收：

- 主窗口每秒更新时间。
- 主窗口可置顶、可拖动、可锁定位置。
- 可创建、编辑、删除、完成多个任务。
- 任务按约定排序。
- 截止倒计时和超时状态正确。
- 手动进度能保存并刷新显示。
- 关闭窗口后程序进入托盘后台。
- 托盘菜单能显示/隐藏主窗、打开设置和退出。
- 开机自启设置能正确写入和移除。
- 临近截止和到期通知只发送一次。
- 重启程序后任务、设置、窗口位置能恢复。

UI 验收：

- 文字在深色背景上可读。
- 优先级同时使用文本和色条表达。
- 时间和倒计时刷新时布局不跳动。
- 图标按钮有 tooltip。
- 低干扰模式下无持续闪烁动画。
- 主窗口在常见 Windows 缩放比例下内容不重叠。

打包验收：

- `scripts/build.ps1` 可生成 `dist/FloatingTODO/`。
- 在无开发环境的 Windows 机器上可直接运行 exe。
- 发布文件夹复制到其他路径后仍能启动并读写本地 `data/`。

## 13. 后续可扩展方向

- 从 JSON 升级到 SQLite，以支持历史统计和归档查询。
- 增加每日/每周完成趋势。
- 增加番茄钟模式。
- 增加任务模板。
- 增加导入导出。
- 增加浅色主题。
